# feat(net): wire ControlledBy + use ActionState query in apply_input_to_player

服务端:
- `replicate_player_for_connected` 给 sim player 同时挂 `Replicate::manual(vec![client_of_entity])` 跟
  `ControlledBy { owner: client_of_entity, lifetime: Persistent }`。
  lightyear 0.26 buffer.rs:537 检查 `controlled_by.owner == sender_entity` 命中后
  `prepare_typed_component_insert(Controlled)` 推 Controlled marker 给 client。
  之前 server 端有 sim player 但没推 Controlled 给 client → client 端 InputPlugin 的
  `input_buffer_query` (With<S::Marker>) 找不到 entity → 发的 InputMessage 内 inputs 永远空。
- `apply_input_to_player` 改用 `Query<(&ActionState<PlayerAction>, &mut Transform, &mut PlayerPos),
  (With<Name>, With<ControlledBy>)>` 取代 `Option<Res<ActionState>>` (ActionState 是 per-entity
  component, 不是 resource, 之前那条路永远 None)。每 tick 30Hz 读 controlled entity 的
  ActionState (lightyear_inputs 0.26 `receive_input_message` server.rs:261 在 input 到达时自动
  attach `InputBuffer + ActionState` 到 query.get_mut 命中的 entity, controlled_by 命中的
  entity 一定有 ActionState), 推 Transform.translation + PlayerPos。

客户端:
- `collect_keys_to_action_state` filter 改 `With<Controlled>` 取代 `With<Player>` (Player 是
  camera marker, 不是 player entity)。`With<Controlled>` 找 server 推过来的镜像 player entity
  (server 端 ControlledBy 加了后, lightyear 自动推 Controlled marker 给 client), 把键盘 W/A/S/D
  写到它的 ActionState, lightyear_inputs client `buffer_action_state` 在 FixedPreUpdate 把
  ActionState 写到 InputBuffer, `prepare_input_message` 在 PostUpdate 序列化成 InputMessage 上行。
- `w_active = w_pressed || cfg.auto_walk` — auto-demo 模式强制按 W (推 input 上行, 让 server
  sim player 移动)。

验证:
- server 端 `lightyear_inputs::server: received input message tick=... client_id=... action=PlayerAction`
  持续 60Hz 跑, server 端 receive 链路 100% 通 ✓
- server 端 sim player `apply_input_to_player` system 现在能找到 ActionState (controlled entity
  query 命中), 但 `message.inputs=[]` 持续空 ✗
- client 端 `MessageReceiver<UpdatesMessage>` 实际 receive 0 条, `replicated=0, playerpos=0`
  持续 — lightyear 0.26 client 端 UpdatesMessage receive 链整体不工作
- client 端 `prepare_input_message` 的 input_buffer_query 一直空, `Number of input messages
  to send: 0` (累积) + `EmptyInputMessage<PlayerAction>` 持续上行
