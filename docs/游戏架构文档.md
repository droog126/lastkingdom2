# 万国起源：最后一国 钻石版 — 游戏架构文档

> **项目名称**：万国起源：最后一国 钻石版
> **游戏类型**：体素沙盒 + 策略模拟
> **技术栈**：Rust + Bevy 0.18.1 + ECS 架构
> **文档版本**：v1.0
> **更新日期**：2026-06-06

---

## 一、项目愿景

**目标**：打造一个"能自我进化"的体素游戏 Demo — AI 自动运行 → 截图 → 读结果 → 改代码 → 重建 → 再运行，形成无人干预的闭环迭代。

**远景**：从"能跑"演进到"百万实体级现代架构"，实现：
- 60 fps 稳定渲染
- 可热插拔扩展
- 可重放可同步

### 2.3 架构差距分析

| 维度 | 当前实现 | 现代架构 | 性能瓶颈 |
|------|----------|----------|----------|
| **数据布局** | `Vec<BlockType>` 稠密数组 | Chunk 分块 + 列式存储 | 全量扫描 96³ ≈ 88 万次访问 |
| **渲染** | 每方块一个 Entity（3000个） | Chunk Mesh（每个 Chunk 1 个 Draw Call） | 移动时帧时间翻倍 |
| **并行** | 1 Hz 串行 Tick | Job System + 任务图 | 8 核只用 1 核 |
| **确定性** | `rand::thread_rng()` 不可重放 | PCG/SplitMix + 固定点数学 | 不可重放、不可同步 |
| **物理** | avian3d 已引入未启用 | 刚体碰撞 + 重力 | 物理碰撞未真用 |
| **扩展** | 无插件机制 | 热插拔 System 注册 | 扩展门槛高 |

## 四、核心子系统详解

### 4.1 世界系统 (World)

**职责**：管理 3D 体素世界的生成、存储、查询

**核心数据结构**：
```rust
// 体素世界
struct World {
    blocks: Vec<BlockType>,  // 稠密数组 (未来改为 Chunk 分块)
    size: i32,               // 世界尺寸 (当前 96)
}

// 方块类型 (12 种)
enum BlockType {
    Air,           // 空气
    Dirt,          // 泥土
    Stone,         // 石头
    Sand,          // 沙子
    Snow,          // 雪
    Leaves,        // 树叶
    Water,         // 水 (半透明)
    Wood,          // 木头
    IronOre,       // 铁矿
    SunstoneOre,   // 阳炎石 (沙漠)
    FrostcoreOre,  // 霜心晶体 (苔原)
    LivingRoot,    // 活根 (丛林)
    BerryThicket,  // 浆果丛
}

// 生物群落 (3 种)
enum Biome {
    Desert,   // 焦土沙漠 → 阳炎石
    Tundra,   // 冰封苔原 → 霜心晶体
    Jungle,   // 繁盛丛林 → 活根
}
```

**世界生成流程**：
```
┌──────────────────────────────────────────────────────────────┐
│                    世界生成流程                              │
├──────────────────────────────────────────────────────────────┤
│ 1. 高度图生成                                               │
│    ├─ 双八度 noise (大尺度山脉 + 小尺度起伏)                  │
│    ├─ 出生点 10 格半径平坦区域                               │
│    └─ 生物群落专属高度偏移 (沙漠-2, 苔原+3)                    │
│                                                              │
│ 2. 地形填充                                                  │
│    ├─ 地表层 (沙/土/雪)                                      │
│    ├─ 中间层 (土, 3 格厚)                                    │
│    └─ 底层 (石头)                                             │
│                                                              │
│ 3. 洞穴生成                                                  │
│    └─ 3D noise > 0.65 处挖空                                 │
│                                                              │
│ 4. 水体填充                                                  │
│    └─ 海平面以下空腔填水                                      │
│                                                              │
│ 5. 矿石生成                                                  │
│    ├─ 各群落专属矿石 (cluster 聚类, 间距≥6)                   │
│    └─ 通用铁矿 (散撒)                                        │
│                                                              │
│ 6. 植被生成                                                  │
│    ├─ 丛林: 高大阔叶树 (5-7 木干 + 3×3×2 树冠)                │
│    ├─ 苔原: 针叶树 (4-5 木干 + 2×2×2 树冠)                   │
│    ├─ 沙漠: 仙人掌                                          │
│    └─ 丛林: 浆果丛                                          │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 AI 观察系统 (AI)

**职责**：Tick-level 闭环 Debug 与异常检测

**设计目标**：
1. **快照** — 每 tick 完整状态序列化
2. **决策日志** — AI 决策 + 上下文 + 结果
3. **不变量断言** — 资源守恒、计数一致、边界检查
4. **异常检测** — 震荡、卡顿、跳变
5. **重放** — 从快照恢复历史状态

**Tick 生命周期**：
```
┌─────────────────────────────────────────────────────────────┐
│                  TickObserver 生命周期                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐   │
│  │  begin_tick │───►│ sim systems │───►│ end_tick    │   │
│  │  记录开始    │    │ (怪物/资源   │    │ 快照+断言   │   │
│  │  时间戳      │    │  /国家 tick)│    │ +异常检测   │   │
│  └─────────────┘    └─────────────┘    └──────┬──────┘   │
│                                                 │           │
│                                                 ▼           │
│                                         ┌─────────────┐   │
│                                         │ report()    │   │
│                                         │ 输出报告    │   │
│                                         └─────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**不变量检查清单**：
| 检查项 | 验证内容 | 违例处理 |
|--------|----------|----------|
| 资源守恒 | current ≤ max, 审计平衡 | 记录违例 tick |
| 怪物计数 | current = Σ(nest.individuals) | 重新计算纠正 |
| 国旗上限 | flag_count ≤ 8 | 拒绝新国旗 |
| 玩家边界 | 位置在世界范围内 | 纠正位置 |
| Tick 时长 | < 50ms | 记录卡顿 |

**异常检测规则**：
| 异常类型 | 检测条件 | 严重度 |
|----------|----------|--------|
| 决策震荡 | 同 AI 连续 5 tick 相同决策 | Warning |
| Tick 卡顿 | 单 tick > 50ms | Error |
| 资源跳变 | 资源突变 (未通过 Sub/Add/Regen) | Warning |
| 结构异变 | Kingdom/Nest 数量异常变化 | Warning |
| 国家瞬灭 | 国旗数突然归零 | Critical |

### 4.6 国家系统 (Nation)

**职责**：国家建立、资源消耗、国旗管理

**核心规则**：
- 每面国旗消耗递增的 Soul：第 N 面 = 20 × N
- 国旗上限：8 面
- 单人模式玩家 ID 固定为 0

**建立流程**：
```
玩家按 F → 检查是否已有国家
              │
              ├─ 已有 → 提示"已经是国王"
              │
              └─ 没有 → 计算成本
                        │
                        ▼
                  扣除 Soul (force_sub)
                        │
                        ▼
                  创建 NationId
                        │
                        ▼
                  player.nation_id = Some(id)
                        │
                        ▼
                  nations.flag_count += 1
```
