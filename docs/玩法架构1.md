# 万国起源 · 玩法架构
## 设计理念：世界即规则，演化即玩法

---

## 一、核心设计哲学

### 1.1 世界优先于玩家

服务端的世界是一个**自运行的演化系统**。无论是否有客户端连接，世界都在按照物理法则和生态规则自行运转。

- **没有"刷怪"**：所有生物都有出生、成长、繁殖、死亡的生命周期
- **没有"刷新"**：资源不会凭空出现，只会从一种形态转化为另一种形态
- **没有"重置"**：世界状态是连续的，不可逆的（除非使用时间倒带）

### 1.2 资源守恒定律

```
世界总质量 = 常量（初始设定）
任何变化 = 形态转化，而非创造/消灭

示例转化链：
阳光 + 水 + 土壤 → 小麦（植物生长）
小麦 → 面包（加工）
面包 → 食物能量（消耗）
食物能量 → 灵魂（生物死亡）
灵魂 → 国家令牌（政治行为）
国家令牌 → 国家旗帜（建国）
国家旗帜被摧毁 → 灵魂释放 → 回归资源池
```

### 1.3 涌现式玩法

设计者只定义**底层规则**，玩法由规则**自发涌现**：

| 设计者定义 | 玩家体验到 |
|-----------|-----------|
| 狼吃羊，羊吃草 | 过度放牧 → 草原退化 → 沙尘暴 |
| 沙尘暴降低能见度 | 沙尘暴掩护突袭 / 沙尘暴中迷路 |
| 河流从高向低流 | 筑坝截流 → 上游洪水 / 下游干旱 |
| 火蔓延需要燃料 | 防火带策略 / 火攻战术 |

## 二、世界分层架构

### 2.1 维度（Dimension）= 独立服务端

每个维度是一个**独立的自运行世界实例**，有自己的：
- 物理规则（重力、时间流速）
- 生态参数（温度、湿度、季节）
- 资源分布（矿物、生物群系）
- Tick 速率（可配置）

```
主世界（Overworld）     Tick = 20Hz，标准重力
├─ 焦土沙漠            高温，阳炎石产地
├─ 冰封苔原            低温，霜心晶体产地
├─ 繁盛丛林            高湿，活根产地
└─ 以太界              Tick = 10Hz，周期性重力扭曲
```

### 2.2 区域（Region）= 空间分区

维度内按 512×512 区块划分为区域：
- 每个区域有独立的生态参数
- 区域间通过"生态边界"交互（动物迁徙、气候传播）
- 玩家不在线时，区域以简化模式运行（关键状态保留，细节休眠）

### 2.3 区块（Chunk）= 最小模拟单元

16×16×16 的体素单元：
- 方块数据（连续内存）
- 实体列表（动态）
- 局部生态状态（土壤湿度、温度）

## 三、时间系统

### 3.1 多层级时间

| 层级 | 单位 | 用途 |
|------|------|------|
| **Tick** | 50ms | 物理、AI、逻辑计算 |
| **秒** | 20 Ticks | 玩家感知 |
| **分** | 60 秒 | 生态循环（生长、繁殖） |
| **时** | 60 分 | 昼夜交替、天气变化 |
| **日** | 24 时 | 季节推进 |
| **季** | 90 日 | 大规模生态变化 |
| **年** | 4 季 | 文明演进 |

### 3.2 时间流速差异

不同区域可以有不同的 Tick 倍率：

```rust
struct TimeDilation {
    region_id: RegionId,
    multiplier: f32,  // 0.1 = 慢10倍，10.0 = 快10倍
    reason: TimeDilationReason,  // 魔法、科技、自然现象
}
```

**玩法示例**：
- 农场区域：10x 加速，作物快速生长
- 主城区域：1x 正常，社交不受影响
- 以太界：0.5x + 周期性重力反转

### 3.3 时间倒带与分支

```
主时间线: T0 ── T1 ── T2 ── T3 ── T4 ── T5 ── T6
                │
                └── 分支 A: T3' ── T4' ── T5'（实验）
                        │
                        └── 合并回主时间线（选择保留）
                        └── 废弃（释放资源）
```

**实现**：
- 每 Tick 记录"状态差异快照"（只记录变化）
- 倒带时回滚到目标 Tick，重放后续输入
- 分支创建时 Copy-on-Write，共享未变化数据

## 四、生态系统设计

### 4.1 能量流动模型

```
        阳光
          │
          ▼
    ┌─────────┐
    │  生产者  │  草、树、作物
    │ (植物)   │
    └────┬────┘
         │ 化学能
         ▼
    ┌─────────┐
    │  初级消费者 │  羊、兔、牛
    │ (食草动物)  │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  次级消费者 │  狼、豹
    │ (食肉动物)  │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  分解者   │  真菌、细菌
    │          │  回归土壤
    └─────────┘
```

### 4.2 物种属性

```rust
struct Species {
    id: SpeciesId,
    name: String,

    // 代谢
    energy_consumption_per_tick: f32,  // 每 Tick 消耗能量
    energy_storage_max: f32,           // 最大储能

    // 繁殖
    reproduction_energy_threshold: f32, // 繁殖所需能量
    reproduction_cooldown: Duration,    // 繁殖冷却
    offspring_count: Range<u32>,        // 后代数量

    // 行为
    diet: Diet,                        // 食性
    territory_size: f32,              // 领地范围
    social_structure: SocialStructure, // 独居/群居

    // 环境适应
    preferred_temperature: Range<f32>,
    preferred_humidity: Range<f32>,

    // 寿命
    lifespan: Duration,
    aging_rate: f32,
}
```

### 4.3 涌现式生态事件

| 触发条件 | 涌现结果 | 玩家应对 |
|---------|---------|---------|
| 狼群密度 > 阈值 | 狼群分裂，新狼群迁徙 | 追踪新狼群位置 |
| 草原过度放牧 | 土壤侵蚀，沙尘暴 | 轮牧、种植固沙植物 |
| 河流筑坝 | 上游湿地形成，下游干旱 | 权衡利弊 |
| 火山喷发 | 短期毁灭，长期肥沃土壤 | 疏散、灾后重建 |
| 物种入侵 | 本地物种灭绝 | 生物防治 |

## 五、文明系统（国家）

### 5.1 国家生命周期

```
流亡者（无国家）
    │
    ▼ 收集灵魂 → 购买令牌
建国者（放置旗帜）
    │
    ▼ 招募成员
部落（5人）
    │
    ▼ 升级建筑
城邦（15人）
    │
    ▼ 科技研发
王国（30人）
    │
    ▼ 建造传送门
帝国（50人）
    │
    ▼ 统一世界 / 被摧毁
胜利 / 灭亡 → 灵魂释放 → 新循环
```

### 5.2 国家内部结构

```
国王（唯一）
├─ 宰相（管理内政）
│   ├─ 财政大臣（资源调配）
│   ├─ 军事大臣（战争指挥）
│   └─ 外交大臣（条约谈判）
├─ 将军（指挥军队）
│   └─ 队长（小队指挥）
└─ 平民（生产、战斗）
    ├─ 矿工、樵夫、农夫（资源生产）
    ├─ 商人（物流、贸易）
    └─ 战士（防御、进攻）
```

### 5.3 外交系统

```rust
enum DiplomaticRelation {
    War,           // 战争：可互相攻击
    Hostile,       // 敌对：无条约，紧张
    Neutral,       // 中立：默认状态
    Trade,         // 贸易：可互通市场
    Alliance,      // 同盟：共享视野、协防
    Vassal,        // 附庸：进贡、受保护

struct DiplomaticAction {
    from: NationId,
    to: NationId,
    action: ActionType,      // 宣战、求和、结盟、断交
    cost: ResourceCost,      // 外交行动消耗
    cooldown: Duration,     // 冷却时间
    betrayal_penalty: f32,  // 背信代价（信誉损失）
}
```

## 六、经济系统

### 6.1 三层经济

| 层级 | 范围 | 媒介 | 特点 |
|------|------|------|------|
| **物物交换** | 个人间 | 直接物品 | 无通胀，效率低 |
| **国家市场** | 国家内 | 金币 | 税收、调控 |
| **世界市场** | 全球 | 稀有资源 | 战略物资 |

### 6.2 物流系统

```
生产点（矿洞/农场）
    │
    ▼ 采集
临时存储（背包/箱子）
    │
    ▼ 商队运输（物理移动，可被劫）
国家仓库
    │
    ▼ 加工/制造
成品
    │
    ▼ 分配
前线 / 市场 / 建筑
```

### 6.3 通货膨胀控制

- **资源总量固定**：没有无限印钞
- **损耗机制**：工具磨损、食物腐烂、建筑老化
- **回收机制**：拆解建筑回收 30-50% 材料
- **世界市场调节**：中立商人高价收购/低价抛售

## 八、终局系统

### 8.1 以太界（终局维度）

进入条件：
- 国家等级 ≥ 帝国
- 建造位面传送门（消耗巨量资源）
- 传送门可被攻击/摧毁

环境特点：
- 重力周期性扭曲（60-120 秒切换）
- 以太幽魂（高机动飞行怪物）
- 虚空精华（终极资源）

### 8.2 神器系统

```rust
struct Artifact {
    id: ArtifactId,
    name: String,
    components: Vec<Component>,     // 合成材料
    effects: Vec<ArtifactEffect>,   // 效果列表
    durability: u32,              // 耐久度
    soul_bind: Option<PlayerId>,  // 灵魂绑定
}

enum ArtifactEffect {
    TimeDilation { radius: f32, multiplier: f32 },  // 时间扭曲
    GravityControl { direction: Vec3, strength: f32 }, // 重力操控
    Resurrection { cooldown: Duration },              // 复活
    MindControl { max_targets: u32, duration: Duration }, // 心灵控制
    WorldEdit { radius: f32 },                        // 世界编辑
}
```

### 8.3 胜利条件

| 胜利类型 | 条件 | 奖励 |
|---------|------|------|
| **征服胜利** | 摧毁所有敌对国家旗帜 | 王冠世界持有权 |
| **科技胜利** | 解锁终极科技"世界之心" | 自定义物理规则 |
| **生态胜利** | 建立可持续生态循环（100年） | 永恒春天 |
| **外交胜利** | 所有国家自愿臣服 | 世界政府 |
| **逃逸胜利** | 建造星际飞船离开世界 | 新维度创建权 |

## 九、玩家体验流程

### 9.1 新手期（0-2小时）

```
出生 → 生存（采集/建造）→ 发现其他玩家 → 
选择：独狼 / 加入国家 → 学习基础职业 → 
参与第一次小规模冲突
```

### 9.2 成长期（2-20小时）

```
专精职业 → 建立/加入国家 → 参与国家建设 → 
学习高级技能 → 参与国战 → 理解经济/外交
```

### 9.3 领袖期（20-100小时）

```
成为国王/大臣 → 制定国家战略 → 指挥大规模战争 → 
建造传送门 → 探索以太界 → 争夺神器
```

### 9.4 传奇期（100+小时）

```
统一世界 / 建立持久和平 → 王冠世界经营 → 
创造自定义内容 → 成为服务器历史的一部分
```

## 十、反作弊与公平性

### 10.1 确定性验证

- 所有客户端运行相同逻辑验证关键操作
- 服务端权威 + 客户端预测回滚
- 异常输入自动标记审查

### 10.2 经济审计

- 每笔资源变动记录审计日志
- 自动检测异常交易（超量、超频）
- 资源守恒实时校验

### 10.3 行为分析

- AI 检测异常行为模式（透视、瞬移、秒杀）
- 玩家举报 + 自动标记系统
- 观战席延迟 + 社区监督
